# 📅 Tutka Tamka – Backend

Серверна частина системи онлайн-бронювання для СТО. Забезпечує обробку запитів, роботу з базою даних та бізнес-логіку системи.

---

### ⚙️ Функціонал

* **REST API:** Обробка запитів від фронтенд-частини.
* **Керування бронюваннями:** Створення, отримання та валідація записів.
* **Робота з БД:** Збереження даних про послуги, клієнтів та графік.
* **Middleware:** Обробка помилок та перевірка вхідних даних.

### 🛠 Технології

* **Node.js**
* **Express.js**
* **MySQL** (через mysql2)
* **Dotenv** (конфігурація середовища)

---

### 📁 Структура проекту

```text
src/
 ├── config/      # Налаштування БД та середовища
 ├── controllers/ # Обробники запитів (логіка маршрутів)
 ├── models/      # Схеми таблиць бази даних
 ├── routes/      # Визначення ендпоінтів (API routes)
 ├── services/    # Бізнес-логіка та взаємодія з БД
 └── utils/       # Допоміжні функції (валідатори, хешери)
app.js            # Налаштування Express та middleware
server.js         # Точка входу (запуск сервера)

🚀 Локальний запуск
1. Клонування репозиторію:
git clone https://github.com/REFRAZ1/sto-booking-backend.git
cd sto-booking-backend

2. Встановлення залежностей:
npm install

3. Налаштування оточення:
Створіть файл .env у кореневій папці та додайте ваші дані (приклад):
PORT=5000
DB_HOST=localhost
DB_USER=root
DB_PASS=your_password
DB_NAME=tutka_tamka_db

4. Запуск:
Звичайний запуск
npm start

Режим розробки (якщо встановлено nodemon)
npm run dev

```
